11/**
12 * An immutable Monero amount, stored exactly as an integer number of atomic units.
13 *
14 * Monero's smallest unit is the *atomic unit* (colloquially "piconero"): 1 XMR =
15 * 10^12 atomic units. monerod reports and accepts amounts as **unsigned 64-bit**
16 * integers of atomic units.
17 *
18 * PHP's `int` is *signed* 64-bit (max ~9.22e18). Monero's cumulative emission is
19 * already ~1.84e19 atomic units, so fields such as `already_generated_coins` cannot
20 * be represented by PHP `int` at all — and `json_decode()` silently degrades an
21 * out-of-range integer to a lossy `float` unless decoded with `JSON_BIGINT_AS_STRING`.
22 * Using `int` for currency is therefore a live precision bug, not merely a style
23 * choice. This class wraps {@see BigInteger} (backed by ext-bcmath, or GMP when
24 * available) to represent the full uint64 range — and beyond — exactly.
25 *
26 * A dedicated value object (rather than a bare {@see BigInteger} or brick/money) is
27 * used because there is a single currency here, and a domain type reads better in
28 * signatures and prevents accidental mixing of atomic-unit and XMR-denominated
29 * numbers.
